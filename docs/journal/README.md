# 作業記録

このフォルダには、日ごとの実装・作業内容を `[日付].md` 形式で記録する。

例: `2026-06-27.md`

要件定義・実装計画（`requirements.md` / `implementation-plan.md`）とは別に、「今日何をしたか」を残すためのログ。

各日のファイルには、AI による実装内容に加えて **自分が確認・学習したこと** も書く。

## 一区切りついたときの記録（運用ルール）

チャットで **作業終了**・**休憩**・**一区切り** など、セッションが区切られたときは、その日の `[日付].md` に **ユーザーがしたこと** をまとめて追記する。

記録する内容の例:

- 動作確認したこと（`npm run dev`、開いた URL など）
- コードを読んで理解したファイル・ポイント
- 疑問を解決した内容
- コード読解の進捗（どこまで理解済みか）

ファイルがまだない日は `YYYY-MM-DD.md` を新規作成する。
